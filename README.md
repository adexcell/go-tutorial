## Проект: "Delayed Notifier"

### Суть: Пользователь может отправить запрос: «Отправь мне уведомление (email/push) через 10 минут/2 часа/3 дня».
---
## Как в этом проекте задействован стек:
1. Postgres (Основная база):
    - Хранение пользователей .
    - Хранение истории уведомлений и их статусов (отправлено/ошибка).
2. Redis (Кэш и Скорость):
    - Кэширование профиля: Чтобы при каждом запросе не ходить в Postgres за данными юзера, будем хранить их в Redis.
    - Rate Limiting: Ограничение количества запросов (например, не более 5 уведомлений в минуту для одного юзера), чтобы нас не заспамили.
3. RabbitMQ (Очередь задач):
    - Когда наступает время отправки, сервис кидает задачу в RabbitMQ.
    - Отдельный воркер (Consumer) забирает задачу и имитирует отправку письма. Это гарантирует, что если сервис отправки писем притормозит, наш основной API не "ляжет".
4. Kafka (Событийная шина / Аналитика):
    - Каждое важное событие (регистрация, создание уведомления, успешная доставка) будем гнать в Kafka.
    - Это нужно для системы аналитики или других микросервисов, которые захотят "подписаться" на наши события в будущем.
5. Viper, Zerolog, Gin:
    - Это наш каркас (инфраструктура), который обеспечивает гибкость, крутые логи и быстрый HTTP.
